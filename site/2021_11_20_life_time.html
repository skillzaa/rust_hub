Life Time

2021-11-20

We can use 'static when we want to indicate that a variable should live as long as the programme its self. However this does not automatically does that.

  • The issue of life times is all about returning references from a function. Since when a reference is originated in a function we return the reference but we need to tell the compiler to keep it alive even after the function has ended. For this we assign life-times to the incomming references and assign the same life-time to the returned ref. The purpose of life time is if we have reference type as incomming arguments and ref type return value. We need to tell Rust to keep the return ref as long as one omf the incomming arguments
